# Agents Don't Read Errors

Counterfactual evaluation of error-message reading in tool-using LLM
agents. Companion repository for the paper *"Agents Don't Read Errors:
Counterfactual Scrambling for Testing Error-Message Use in Tool-Using
LLMs"*, submitted to the NeurIPS 2026 Evaluations and Datasets Track.

We test four open-weight models (Mistral-7B, Llama-3.1-8B, Qwen2.5-14B,
Qwen3-14B) on 100 MBPP-sanitized tasks per cell across four error-content
conditions (Real, Scrambled, Misleading, Empty). We measure whether
agents recover from errors at different rates when error content is
preserved versus destroyed.

**Headline results** (paired Δ in error recovery rate, real vs. scrambled,
Holm-corrected over four model tests):

| Model         | n  | ΔERR     | 95% BCa CI         | p_Holm   |
|---------------|----|----------|--------------------|----------|
| Mistral-7B    | 95 | −1.05 pp | [−7.37, +2.11]     | 0.655    |
| Llama-3.1-8B  | 97 | +1.37 pp | [−2.23, +4.98]     | 0.527    |
| Qwen2.5-14B   | 90 | +4.81 pp | [+0.37, +10.37]    | 0.062    |
| Qwen3-14B     | 36 | +18.06 pp | [+6.94, +30.56]   | **0.033** |

Only the reasoning-tuned Qwen3-14B reads errors after correction, and
it does so on only 42% of trials. See the paper for full discussion.

## Quick start

Reproduce all paper results from the released JSONL files:

```sh
cd code/
python3 -m pip install -r requirements.txt
python3 analyze.py        # produces results_summary.json + merged_records.pkl
python3 build_figures.py  # produces ../figures/*.{pdf,png} and ../paper/tables/*.tex
```

The output of `analyze.py` matches the per-cell numbers in the paper
exactly. The output of `build_figures.py` reproduces Figures 1-4.

## Data schema

Each line of each `data/*.jsonl` file is one agent trial:

```json
{
  "status": "ok",
  "context": {
    "dataset": "mbpp",
    "model": "qwen3_14b",
    "scaffold": "react_native",
    "condition": "real",
    "task_id": "mbpp-130"
  },
  "metrics": {
    "task_success": 0.0,
    "err": 0.0,
    "ecr": 0.0,
    "n_errors": 2,
    "n_recoveries": 0,
    "total_tokens": 4061,
    "wall_time_seconds": 17.4
  },
  "trajectory": {
    "turns": [
      {"kind": "system",   "content": "..."},
      {"kind": "user",     "content": "## Task..."},
      {"kind": "assistant","content": "<think>..."},
      {"kind": "tool_call","tool_name": "submit_code", "tool_args": {"code": "..."}},
      {"kind": "tool_error","content": "Traceback...",
                            "original_error": "Traceback...",
                            "substituted": false}
    ],
    "final_answer": "...",
    "tool_call_count": 2,
    "error_count": 2
  }
}
```

`tool_error` turns include both the substituted content (what the
agent sees) and the original error (what the underlying environment
produced). When `condition=real`, these are identical and
`substituted=false`. For other conditions, `content` contains the
post-substitution text and `substituted=true`.

The paper's primary analysis uses MBPP records only and the Qwen3-14B
records from the 8,192-token replication run. The 1,024-token Qwen3
records are present in the original sweep file for completeness but
excluded from headline statistics; see Appendix A of the paper.

## Re-running the experiment

The agent harness that produced these trajectories (Ollama client,
ReAct scaffold, condition transformers) is not in this repository:
the released trajectories together with `code/analyze.py` are
sufficient to verify all quantitative claims in the paper without
re-running inference. To regenerate trajectories you would need:

- A running Ollama server with the four models pulled
  (`ollama pull mistral:7b-instruct llama3.1:latest qwen2.5:14b qwen3:14b`).
- Approximately 12 hours of inference time on a 24 GB GPU.
- The harness, available on request after de-anonymization.

Inference is deterministic at temperature 0; trajectories should
reproduce bit-identically given the same model checkpoints and prompt
templates.
